### 3. 性能优化
重点优化 OCR 阶段：
- `OcrEngine` 按 `(languages, gpu)` 缓存 `easyocr.Reader`，避免重复加载模型
- OCR 识别改为直接传入 `numpy.ndarray`，避免临时文件 I/O
- `TfidfVectorizer` 改为使用 jieba 中文分词器

性能对比（样例文档）：
- 首次校对：~3.5s → ~2.45s
- 缓存后再次校对：~3.5s → ~0.22s（10 倍以上提升）

### 4. 功能增强

#### 4.1 OCR 段落定位
- `docx_parser.py` 通过 `doc.inline_shapes` 与 XML 祖先段落定位，记录图片所在 `block_index`
- `OcrIssue` 新增 `context_block` 字段
- UI 问题报告中展示图片所在段落上下文

效果示例：
```text
截图 #0（位于「3. 系统截图...」之后）OCR 文字覆盖需求关键词较少...
```

#### 4.2 表格内容比对
新增 `checkers/table_checker.py`：
- 按表头语义 + 行键重叠度匹配需求表格与投标表格
- 检查行数/列数不一致
- 逐格比较单元格内容
- 检测缺失或多余的表格

集成：
- `ProofreadResult` 新增 `table_issues`
- UI 增加表格问题展示
- 新增 `tests/test_table_checker.py`

#### 4.3 多时间/多参数支持
重构 `consistency_checker.py`：
- 7×24 小时服务单独处理，避免与普通时间提取重复
- 支持从单条需求中提取多个时间值
- 每个时间值根据其前方约束词独立判断比较方向（≥ / ≤）
- `_compare_time()` 返回消息列表，可生成多个 `TIME_MISMATCH` issue

新增 `tests/test_consistency_multi.py` 覆盖：
- 单时间值
- 多时间值
- 7×24 小时
- 多参数（次/人）

效果示例：
```text
[time_mismatch] 要求不少于 3.0年，投标仅 2.0年
[time_mismatch] 要求不超过 6.0个月，投标为 8.0个月
```
